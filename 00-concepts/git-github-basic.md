# Git & GitHub Basics

### - *Version Control System (VCS) :* ### 
A system that helps track changes in files and lets multiple people work on the same project safely.

### - *Git :*
Git is a tool that helps you track changes in your code and manage different versions of a project.

### - *GitHub :* 
GitHub is a website where you can store Git repositories and work together with others.

### - *Repository :*
A repository is a project folder that contains your code and its history.

### - *Local Repository :*
A repository that exists on your computer.

### - *Remote Repository :*
A repository stored on the internet (for example, on GitHub).

### - *Clone :*
Clone means copying a remote repository to your local computer.

### - *Commit :*
A commit saves your changes with a short message.

### - *Branch :*
A branch is a separate version of the project where you can work without affecting the main code.

### - *Main Branch :*
The main branch contains the stable version of the project.

### - *HEAD :*
HEAD points to the branch you are currently working on.

### - *Staging Area :*
A place where you keep files that are ready to be committed.

### - *Merge :*
Merge means combining changes from one branch into another.

### - *Pull Request (PR) :*
A request to merge your changes into another branch.

### - *Fork :*
A fork is your own copy of someone else’s repository.

### - *Origin :*
Origin is the default name for the remote repository.

### - *Upstream :*
Upstream refers to the original repository from which a fork was created.

### - *.gitignore :*
A file that tells Git which files should not be tracked.

### - *Working Tree :*
The folder where you edit your project files.

### - *Push :*
Send your commits to a remote repository.

### - *Pull :* 
Get the latest changes from a remote repository.

### - *Fetch :*
Download changes without merging them.

### - *Rebase :*
Move your commits on top of another branch’s commits.
