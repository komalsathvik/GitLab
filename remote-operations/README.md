# Remote Operations

This section focuses on working with **remote repositories** such as GitHub, GitLab, or Bitbucket.

After learning local Git basics, remote operations allow you to:

- Share your code with others
- Collaborate in teams
- Sync changes between local and remote repositories

## What You Will Learn

- How to connect a local repository to a remote one
- How to push local commits to GitHub
- How to fetch and pull updates from a remote repository
- How to view commit history from remote branches

## Key Concepts

- **Remote Repository**  
  A version of your project hosted on the internet.

- **origin**  
  The default name Git gives to a remote repository.

- **Push vs Pull**
  - Push → send your commits to remote
  - Pull → bring remote commits to local

## Commands Covered

- `git remote`
- `git push`
- `git fetch`
- `git pull`
- `git log`

## Recommended Order

1. git remote
2. git push
3. git fetch
4. git pull
5. git log
